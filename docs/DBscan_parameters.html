Illustration of our criteria for paramteter estimation of DB-scan by the “elbow” method.

Leuk_18

Recall the workflow used here utilizes dimensionality reduction by 3D-UMAP.

We tested all variants of data sizes in terms of numbers of highly variable genes used that are listed below (article contains selection (500,1000,2000,5000, 10000,20000) due to provided constraints on the space we can utilize).

Method for estimation of the eps parameter seeks an “elbow point” according to the plot of sorted kNN distances of each point, where \(k=2l -1\) where \(l\) is the dimension of the space in which we do clustering (2*3 - 1). Scripts in this repository starting with s1 prefix compute the distances and show them in common axes on a plot in a pdf file, which reveals that we can reasonably use the same criterion for any of the gene numbers used, for illustration purposes. Here we show the case for n_genes=2000:

# kNNdist plot

{plot(kNN_curve, main="kNN distances, UMAP embedding of samples using 2000 genes")
abline(h=0.3)}
